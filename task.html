Writing with DITA Topics:

<task>, <concept>, & <reference>

Topic Types:

  • <task>:
    Tells how to do something, and usually includes numbered steps -- a procedure.

  • <concept>:
    Describes something.

  • <reference>:
    Explains details of interest about things and materials, and usually includes information in tables or lists.

<task>

What is a <task>?

  • Topic dedicated to procedural information
  • <concept> and <reference> topics support task topics, but are written in different .dita files.

<task> best practices

  • One procedure per topic
  • Create subtasks for longer procedures
  • Combine short, sequential steps
  • Every step should use an imperative sentence
  • Every <task> element should have a meaningful ID attribute: <task id="espresso-connect">

Common Task Topic Elements

ELEMENT DESCRIPTION
<task> Main containing element of a task topic; answers the "How to ...?" question.
<taskbody> Contains information specific to completing a task.
<title> Provides the topic title
<shortdesc> Introduces the task
<prereq> Prerequisites that the user needs to know or do before starting the current task
<context> Background information for users to understand task
<steps> Contains ordered steps for the entire procedure
<steps-unordered> Contains unordered steps for the entire procedure
<step> Represents one action in the procedure; contained within <steps> or <steps-unordered>
<cmd> A command: An imperative statement of the <step>; Required.
<substeps> Breaks down a <step> into separate actions; Use sparingly, if more than one nested <substeps>, then rewrite as separate task topic.
<substep> Same structure as <step>. But, a <substep> cannot include nested numbered steps.
<info> Provides additional info about the step, when needed.
<stepresult> What happens when step is completed
<result> Expected outcome when entire task is done
<stepxmp> An example of what happens when step is completed
<example> Example that supports the broader task, rather than specific step.
<choices> > <choice> Displays choices in a bulleted list. Each <choice> is contained by <choices>.
<choicetable> Displays choices in a 2-column table. Describe steps for each choice
<postreq> Info about what must be done after task is completed
<fig> Figure with optional <title>. Can contain range of objects, including text and images.
