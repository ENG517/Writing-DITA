Writing with DITA Topics:

<task>, <concept>, & <reference>

Topic Types:

  • <task>:
    Tells how to do something, and usually includes numbered steps -- a procedure.

  • <concept>:
    Describes something.

  • <reference>:
    Explains details of interest about things and materials, and usually includes information in tables or lists.

<reference>

What is a <reference>?

  • Also supports goals of the task topics
  • A collection of facts, rather than an explanation about necessary conceptual information

<reference> best practices

  • Describe one type of reference information per topic
  • Use noun-based phrases for titles
  • Write and design for scannability
  • Be consistent about design/elements across topics
  • Use the correct type of table (table vs. simple table, vs. properties)
  • Difficulty: Modularize reference info or not. Rule of thumb: If more than 2 pages, break it up, logically.
  • All <reference> elements require a <section> element
  • Do not "paper" (p. 60) your reference topics. Avoid writing topics that the user does not need. (Example: UI icons.)

Common <reference> elements

DITA ELEMENT DESCRIPTION
<title> Provides the topic title
<shortdesc> Introduces the topic
<refbody> Contains the body of the topic
<section> Organizes content in a topic into sections
<table> Contains the table content
<simpletable> Contains the table content that doesn't require a title
<properties> Lists properties in a table by type, value, and description
<ul> Displays a content as an unordered bulleted list
<dl> Displays a list of terms or short concepts and their definitions
<example> Contains example content that explains or supports the topics
<parml> Displays parameters in a format similar to a definition list
